# Contributing to AI Research Assistant

Thank you for your interest in contributing! Here's how you can help.

## Areas for Contribution

- [ ] Support for more file formats (DOCX, HTML, JSON)

- [ ] Streaming responses for long answers
- [ ] Integration with more LLM providers (Ollama, Claude, Llama2)

### Performance

### Documentation
- [ ] Examples and tutorials
- [ ] API documentation

- [ ] Video demos

### Testing
- [ ] Unit tests
- [ ] Integration tests
- [ ] End-to-end tests
- [ ] Performance tests

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Install dev dependencies: `pip install -r requirements-dev.txt`
5. Make your changes
6. Run tests: `pytest`
7. Commit: `git commit -am 'Add feature'`
8. Push: `git push origin feature/your-feature`
9. Create Pull Request

## Code Style

- Follow PEP 8
- Use type hints
- Add docstrings to functions
- Keep functions small and focused

## Commit Messages

Use clear, descriptive commit messages:

```
feat: Add document reranking
fix: Handle empty query results
docs: Update embeddings guide
test: Add search engine tests
```

## Pull Request Process

1. Update README if needed
2. Add tests for new functionality
3. Ensure all tests pass
4. Update documentation
5. Request review

Open an issue or discussion for questions about contributing.
